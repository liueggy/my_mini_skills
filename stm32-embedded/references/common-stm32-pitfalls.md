# STM32 常见踩坑清单

## 时钟与启动
- `SystemClock_Config()` 与芯片实际晶振不匹配。
- 外设时钟未开启就访问寄存器。
- PLL、AHB、APB 分频理解错误，导致串口/定时器频率不对。
- Flash wait state 配置不正确，高主频下容易异常。
- 启动文件、链接脚本、芯片型号不匹配。

## GPIO 与复用
- GPIO 模式配置错误：输入/输出/复用/模拟选错。
- 上下拉缺失导致输入悬空。
- 推挽/开漏配置不符合外设需求。
- 复用功能 AF 编号错误。
- 引脚与开发板实际连线不一致。

## 中断
- 只开了外设中断源，没有在 NVIC 里使能。
- 中断服务函数名和启动文件中的向量名不一致。
- 忘记清中断标志，导致重复进入或根本不进。
- 中断优先级分组与 RTOS 配置冲突。
- 在中断中做阻塞调用、打印、大量计算。

## DMA
- DMA 通道/流/请求映射错误。
- 源地址、目标地址、数据宽度、增量模式不匹配。
- 缓冲区是局部变量，函数退出后地址失效。
- 外设 DMA 请求没开，只配了 DMA 控制器。
- 传输完成/半传输回调没接上。
- Cache/对齐问题（常见于 H7/F7）。

## 串口 / SPI / I2C
- UART 波特率与时钟源不匹配。
- SPI 主从模式、CPOL、CPHA、位宽、片选时序不一致。
- I2C 上拉电阻缺失或过大，导致波形异常。
- I2C 地址 7bit/8bit 混淆。
- 阻塞式通信放在关键流程里导致系统卡顿。

## 定时器
- 预分频和自动重装值计算错，导致周期不对。
- PWM 极性、通道、复用脚不对。
- 输入捕获边沿选错。
- 更新中断未开启或标志未清。
- 高级定时器忘了开主输出（如 BDTR/MOE）。

## RTOS
- 任务栈开太小导致随机死机。
- 中断优先级高于 RTOS 允许范围，调用了不安全 API。
- 用 `HAL_Delay()` 卡住任务调度。
- 共享资源缺少互斥保护。
- 任务优先级设计不合理导致饥饿或反转。

## 内存与 C 语言
- 数组越界、空指针、野指针。
- 共享变量未加 `volatile`。
- `printf` 占栈/耗时过大。
- 结构体对齐与协议帧直接强转引发问题。
- 把复杂业务逻辑写进中断。

## 调试建议
- 先确认硬件连线，再看软件配置。
- 优先检查时钟、GPIO 复用、中断使能、DMA 映射。
- 能上示波器/逻辑分析仪就不要只猜。
- 尽量加入错误码、状态灯、串口日志、断点观察。
- 先做最小闭环验证，再逐步叠加功能。
