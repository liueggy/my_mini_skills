# Obsidian 知识库目录结构说明

## 📁 整体结构

```

├── main/                    # 主知识库（主要工作区）
```

## 📝 笔记命名规范

### 基本原则
1. **清晰明确**：名称能准确反映内容
2. **简洁扼要**：避免过长，控制在20字以内
3. **使用中文**：主要使用中文命名
4. **包含关键词**：在名称中体现核心主题

### 命名模式
- **知识类**：`主题 + 描述`，如 `Linux 学习手册`
- **教程类**：`主题 + 入门/教程/指南`，如 `Docker 入门教程`
- **基础类**：`主题 + 基础`，如 `SSH 基础与远程登录`
- **专题类**：`主题：细分主题`，如 `网络基础：IP、端口、DNS、TCP-UDP`

### 特殊字符使用
- **冒号**：用于主副标题分隔
- **括号**：用于补充说明或英文原名
- **空格**：中文和英文、中文和数字之间加空格
- **避免使用**：`\ / : * ? " < > |` 等文件系统保留字符

## 🏷️ 标签系统

### 标签分类
1. **技术领域**：`Linux`、`嵌入式`、`网络`、`算法`、`数据库`
2. **工具平台**：`Git`、`Docker`、`ROS`、`Python`、`JavaScript`
3. **概念类型**：`基础`、`进阶`、`原理`、`实践`、`优化`
4. **项目状态**：`进行中`、`已完成`、`待整理`、`待验证`
5. **学习阶段**：`入门`、`掌握`、`精通`、`复习`

### 标签使用原则
1. **适度原则**：每篇笔记3-8个标签
2. **层次原则**：从通用到具体
3. **一致原则**：相同概念使用相同标签
4. **组合原则**：使用标签组合精确描述

## 📊 元数据规范

### 必需字段
```yaml
---
分类: 操作系统        # 必须，从固定分类中选择
标签: []              # 必须，标签列表
创建日期: 2026-03-20  # 必须，YYYY-MM-DD格式
最后修改: 2026-03-20  # 必须，与创建日期相同或更新
状态: 已完成          # 必须，进行中/已完成/待完善
---
```

### 可选字段
```yaml
base: "[[计算机知识手册.base]]"  # 基础模板
cover: "[[图片名称.png]]"        # 封面图片
难度: 入门/中级/高级             # 内容难度
预计阅读: 10分钟                 # 预计阅读时间
版本: 1.0.0                     # 文档版本
```

### 状态说明
- **进行中**：内容还在编写或更新中
- **已完成**：内容基本完成，可以阅读使用
- **待完善**：需要补充或优化内容
- **已归档**：不再维护的历史内容

## 🔗 链接管理

### 内部链接
- **双括号语法**：`[[笔记名称]]`
- **带别名**：`[[笔记名称|显示文本]]`
- **标题链接**：`[[笔记名称#章节标题]]`

### 链接原则
1. **相关优先**：优先链接到相关主题笔记
2. **双向链接**：重要的关联建立双向链接
3. **避免死链**：定期检查链接有效性
4. **适度链接**：避免过度链接影响阅读

## 📁 文件组织

### 目录层级
- **一级目录**：主要分类（Linux、嵌入式、工具等）
- **二级目录**：子主题或项目
- **三级目录**：具体教程或专题

### 资产管理
- **图片资产**：放在笔记同级的 `assets/` 目录
- **命名规范**：`主题_序号.扩展名`，如 `STM32_01.png`
- **引用方式**：`![[图片名称.png]]`

### 模板使用
- **知识笔记**：使用 `knowledge-base.md` 模板
- **学习笔记**：使用 `learning-note.md` 模板
- **项目文档**：使用 `project-doc.md` 模板
- **日常笔记**：使用 `daily-note.md` 模板
