# Obsidian 笔记元数据规范

## 📋 概述

本文档定义了 Obsidian 笔记的元数据（YAML Frontmatter）规范，确保知识库的一致性和可维护性。

## 🏗️ 元数据结构

### 基本结构
```yaml
---
# 必需字段
分类: 操作系统
标签: []
创建日期: 2026-03-20
最后修改: 2026-03-20
状态: 已完成

# 可选字段
base: "[[计算机知识手册.base]]"
cover: "[[Linux 学习手册.png]]"
难度: 入门
预计阅读: 15分钟
版本: 1.0.0
相关笔记: []
优先级: 中
---
```

## 📊 字段详细说明

### 1. 分类（必需）
**作用**：笔记的主要分类，用于组织知识库结构

**可选值**：
```yaml
分类: Linux                # Linux 操作系统
分类: 嵌入式               # 嵌入式开发
分类: 工具                 # 开发工具
分类: 机器人               # 机器人技术
分类: 网络                 # 网络技术
分类: 算法                 # 算法与数据结构
分类: 前端                 # 前端开发
分类: 后端                 # 后端开发
分类: 数据库               # 数据库技术
分类: 云计算               # 云计算
分类: 人工智能             # AI/ML
分类: 产品                 # 产品管理
分类: 设计                 # 设计相关
分类: 其他                 # 其他分类
```

**使用规则**：
- 每篇笔记必须有且只有一个分类
- 从固定列表中选择，保持一致性
- 如果现有分类不适用，可创建新分类并更新此文档

### 2. 标签（必需）
**作用**：多维度标记笔记内容，便于搜索和关联

**标签分类**：
```yaml
# 技术领域标签
标签:
  - Linux
  - 嵌入式
  - 网络
  - 算法
  - 数据库
  - 云计算
  - 人工智能

# 工具平台标签
标签:
  - Git
  - Docker
  - ROS
  - Python
  - JavaScript
  - Vue
  - React
  - Node.js

# 概念类型标签
标签:
  - 基础
  - 进阶
  - 原理
  - 实践
  - 优化
  - 调试
  - 部署

# 项目状态标签
标签:
  - 进行中
  - 已完成
  - 待整理
  - 待验证
  - 已归档

# 学习阶段标签
标签:
  - 入门
  - 掌握
  - 精通
  - 复习
  - 备忘
```

**使用规则**：
- 至少包含1个标签，建议3-8个
- 优先使用现有标签，保持一致性
- 标签使用小写或保持原样（如工具名保持原大小写）
- 相关标签可以组合使用，如 `[Linux, 基础, 入门]`

### 3. 创建日期（必需）
**作用**：记录笔记创建时间

**格式**：`YYYY-MM-DD`
```yaml
创建日期: 2026-03-20
```

**使用规则**：
- 必须使用标准日期格式
- 创建后不再修改
- 如果是迁移的旧笔记，使用原始创建日期或迁移日期

### 4. 最后修改（必需）
**作用**：记录笔记最后修改时间

**格式**：`YYYY-MM-DD`
```yaml
最后修改: 2026-03-27
```

**使用规则**：
- 每次修改笔记内容后更新此字段
- 格式与创建日期相同
- 应晚于或等于创建日期

### 5. 状态（必需）
**作用**：标识笔记的完成度和维护状态

**可选值**：
```yaml
状态: 进行中    # 内容还在编写或更新中
状态: 已完成    # 内容基本完成，可以阅读使用
状态: 待完善    # 需要补充或优化内容
状态: 已归档    # 不再维护的历史内容
状态: 草稿      # 初步草稿，未整理
```

**状态说明**：
- **进行中**：正在积极编写或更新
- **已完成**：内容完整，格式规范，可以分享
- **待完善**：有内容但需要优化，或缺少某些部分
- **已归档**：历史版本，不再更新，仅作参考
- **草稿**：初步想法或收集的材料，未整理

### 6. base（可选）
**作用**：指定基础模板

**格式**：
```yaml
base: "[[计算机知识手册.base]]"
```

**使用规则**：
- 主要用于知识类笔记
- 保持与现有笔记一致
- 如果不需要可以省略

### 7. cover（可选）
**作用**：指定封面图片

**格式**：
```yaml
cover: "[[Linux 学习手册.png]]"
```

**使用规则**：
- 图片应放在笔记同目录或 assets 目录
- 使用 `![[图片名称.png]]` 语法引用
- 封面图片应能代表笔记主题

### 8. 难度（可选）
**作用**：标识内容难度级别

**可选值**：
```yaml
难度: 入门    # 适合初学者
难度: 中级    # 需要一定基础
难度: 高级    # 需要深入理解
难度: 专家    # 专业领域深度内容
```

### 9. 预计阅读（可选）
**作用**：估计阅读所需时间

**格式**：
```yaml
预计阅读: 10分钟
预计阅读: 30分钟
预计阅读: 1小时
```

### 10. 版本（可选）
**作用**：文档版本号，用于跟踪更新

**格式**：语义化版本 `主版本.次版本.修订号`
```yaml
版本: 1.0.0
版本: 2.1.3
```

**版本规则**：
- **主版本**：重大更新，可能不兼容
- **次版本**：功能增加，向下兼容
- **修订号**：错误修复和小改进

### 11. 相关笔记（可选）
**作用**：手动指定相关笔记链接

**格式**：
```yaml
相关笔记:
  - "[[Linux 学习手册]]"
  - "[[SSH 基础与远程登录]]"
  - "[[网络基础]]"
```

### 12. 优先级（可选）
**作用**：标识笔记的重要程度

**可选值**：
```yaml
优先级: 高      # 核心知识，必须掌握
优先级: 中      # 重要知识，建议掌握
优先级: 低      # 扩展知识，了解即可
```

## 🔄 元数据更新规则

### 创建新笔记
1. 填写所有必需字段
2. 根据内容选择适当的可选字段
3. 确保格式正确
4. 保存前验证 YAML 语法

### 更新现有笔记
1. 更新"最后修改"日期
2. 如有需要，更新"状态"
3. 更新"版本"号（如有）
4. 添加或修改相关标签
5. 保持其他字段不变

### 批量更新
1. 使用脚本工具进行批量操作
2. 操作前备份重要笔记
3. 先测试单个样本
4. 记录更新日志

    """解析 YAML 前端元数据"""
    if content.startswith("---\n"):
        parts = content.split("---\n", 2)

            yaml_content = parts[1]
            return yaml.safe_load(yaml_content)

## 📈 最佳实践

### 一致性原则
1. **分类一致**：相同主题使用相同分类
2. **标签一致**：相同概念使用相同标签
3. **格式一致**：所有笔记使用相同元数据格式
4. **命名一致**：文件和目录命名规范统一

### 维护性原则
1. **及时更新**：修改内容后立即更新元数据
2. **定期检查**：每月检查元数据完整性
3. **批量处理**：使用脚本处理重复任务
4. **文档更新**：元数据规范变化时更新所有笔记

### 扩展性原则
1. **向后兼容**：新增字段不影响现有笔记
2. **灵活扩展**：允许添加自定义字段
3. **工具支持**：确保工具能处理所有字段
4. **迁移友好**：便于导入导出和迁移

## 🚨 常见问题

### Q1：如何选择分类？
A：根据笔记主要内容选择最相关的分类。如果不确定，参考现有类似笔记的分类。

### Q2：标签太多怎么办？
A：控制在3-8个标签，优先使用核心标签。可以使用标签组合精确描述。

### Q3：日期格式错误？
A：必须使用 `YYYY-MM-DD` 格式，如 `2026-03-20`。

### Q4：状态如何选择？
A：根据笔记完整度选择。正在编写的用"进行中"，完整可用的用"已完成"。

### Q5：可选字段都要填吗？
A：不是。只填写对笔记有意义的字段。保持简洁。
