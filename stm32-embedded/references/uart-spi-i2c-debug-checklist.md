# UART / SPI / I2C 排障清单

## UART 排障

### 现象：串口完全无输出
1. 确认 `USARTx` 时钟已开启。
2. 确认 TX 引脚复用配置正确。
3. 确认波特率计算基于正确的时钟源。
4. 确认串口助手参数一致：波特率、数据位、停止位、校验位。
5. 确认 USB 转串口接线正确，GND 共地。
6. 用最小代码只发送固定字符串测试。
7. 若重定向 `printf`，检查底层发送函数是否真的走到。

### 现象：能发不能收
1. 检查 RX 引脚复用与上下拉。
2. 检查接收中断是否使能。
3. 检查 `HAL_UART_Receive_IT()` / DMA 接收是否真正启动。
4. 检查回调函数是否被调用。
5. 检查是否被其他代码反复重配串口。
6. 检查缓存区是否越界、是否被覆盖。

### 现象：偶发丢包/乱码
1. 检查波特率误差与系统时钟误差。
2. 检查接收缓冲区长度是否足够。
3. 检查中断处理是否过慢。
4. 检查是否需要环形缓冲区。
5. 检查 DMA + 空闲中断方案是否更合适。
6. 检查线缆长度、干扰、电平匹配。

## SPI 排障

### 现象：读写全是 0x00 / 0xFF
1. 检查 SCK/MISO/MOSI/CS 引脚连接。
2. 检查主从模式是否一致。
3. 检查 CPOL/CPHA 是否匹配从设备。
4. 检查片选时序是否正确。
5. 检查数据位宽 8bit/16bit 是否一致。
6. 先用逻辑分析仪看时钟和数据线是否有波形。

### 现象：时好时坏
1. 检查 SPI 时钟是否过高。
2. 检查 CS 拉低和拉高时序。
3. 检查 DMA/中断发送时缓冲区是否提前被改写。
4. 检查是否有多个设备共享总线但片选处理不严谨。

## I2C 排障

### 现象：总线一直忙 / 通信失败
1. 检查 SCL/SDA 是否有上拉电阻。
2. 检查电平是否匹配。
3. 检查地址是 7bit 还是左移后的 8bit。
4. 检查时序配置是否适合当前主频与总线速度。
5. 检查从设备上电时序与初始化时机。
6. 检查 SDA/SCL 是否被某个设备拉死。

### 现象：偶发 NACK
1. 检查寄存器地址和数据顺序。
2. 检查 repeated start 是否符合器件协议。
3. 检查总线速度是否过高。
4. 检查线长、电容负载、干扰。
5. 检查器件是否在忙状态时被访问。

## 推荐验证顺序
1. 最小化代码，只保留一个外设测试。
2. 用示波器/逻辑分析仪看物理波形。
3. 再核对 CubeMX/HAL 配置和手册时序。
4. 最后再怀疑协议解析和上层逻辑。
